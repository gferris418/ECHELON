## Rules for humans and AI contributors

1. Read v0.3 and the relevant earlier ADR or historical section before proposing work.
2. Identify the active phase and acceptance criteria.
3. Produce a plan, risks, rollback plan, and documentation change list.
4. Obtain approval before implementation.
5. Update the current baseline documentation in the same pull request as code.

GitHub is the authoritative source. GitBook is a published, human-readable mirror. If they conflict, GitHub wins.
